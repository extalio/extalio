xNewsletter #1

Going through Twitter and blogs every day and filtering through the junk takes too much time. We figured others are having the same problem. Which is why we decided to compile a list of the interesting content that was published lately. We will publish every few weeks (when enough exciting things have happened).

We would love to hear your thoughts! What was most interesting? What was missing? Any additional updates you would like to receive? You can send feedback by replying to this email (or just talking to us :)).

If you have friends that may enjoy this - they can subscribe on extal.io/xnews to get the newsletter in the email or use our RSS.
Happy hacking in 2019!

35C3

From Zero to Zero Day #madeinisrael

RCE in Microsoft Edge’s ChakraCore found by 18-yo Jonathan from MSRC. A type-confusion in the JIT compiler causes the vulnerability. A JavascriptNativeFloatArray is mistaken for a JavascriptArray by abusing a state mismatch with array’s missing values functionality and type conversion through prototype assignment. This confusion yields an RW primitive which can then be used to overwrite the stack with a ROP chain.

Modern Windows Userspace Exploitation #madeinisrael

Overview of current mitigations, bypasses, and basic exploitation terminology (primitives) with concrete examples based on a CTF. Saar walks us through the exploitation process with increasing mitigations - first on windows7 with LFH, then on windows10 with LFH randomization and CFG, then on windows 10 with ACG, CIG, and a restriction on creating child processes. “The point in life is to execute arbitrary code :)”

Attacking Chrome IPC

The first half of the lecture is about how to get into vulnerability research, how to get better, and the kind of mindset to have when looking for bugs. In the second half is about how to find vulnerabilities in Chrome IPC, giving a good overview of the system with tips on how to fuzz it.

The Layman's Guide to Zero-Day Engineering

This lecture explains how to approach 0day research from nothing to pwn. The covered stages are: Setting expectations, Reconnaissance, Target Selection, Bug Hunting, Exploit Development, Exploit Reliability, and Responsibilities.

CTF

A 48-hour CTF, with more than 600 participating teams, and 30 challenges, in any field of expertise. No one solved some challenges. Pasten team (#madeinisrael) won second place, after winning the first place for 3 consecutive years.
Write-ups on a few challenges: Collection, Newphonewhodis, Sequence.

Vulnerabilities

FreeRTOS TCP/IP Stack Vulnerabilities #madeinisrael

A write-up about multiple vulnerabilities in FreeRTOS TCP/IP stack (in several of the major protocols) causing information leaks, DOS, and RCE.

Adventures in Video Conferencing

A 5-part series is exposing several vulnerabilities with video conferencing applications, such as WebRTC, FaceTime, and Whatsapp. The articles start with an overview of the relevant protocols. In WebRTC (most common VC implementation used) Natashenka focuses on the “signaling” attack surface (before user interactions) - by fuzzing the RTP protocol and modifying chrome. The series provides researchers with the tools and hooks used during the research.

50 CVEs in 50 Days: Fuzzing Adobe Reader #madeinisrael

The Check Point security team discusses the process of finding more than 50 vulnerabilities in JP2KLib.dll (used as a parser by Adobe Reader). They don’t go into details of the vulnerabilities themselves, but rather describe the entire process of looking for them: from setting up the fuzzing environment to coding a wrapper around the DLL, and running WinAFL in optimal conditions.

systemd-tmpfiles root privilege escalation by following non-terminal symlinks

A local privilege escalation bug in the systemd-tmpfiles process. The process makes sure not to operate on symlinked files but does not protect against a case where the symlink is part of the directory.

On VBScript

Found 3 vulnerabilities in VBScript, using grammar fuzzing. One allows writing to freed memory. The seconds allows an OOB read. And the last one causes a reference leak that can trigger a UAF bug. All 3 vulnerabilities are relatively simple.

Interesting Research

First Steps in Hyper-V Research #madeinisrael

Overview of the Hyper-V stack and introduction to how to look for vulnerabilities there.

Searching statically-linked vulnerable library functions in executable code

Identifying compiled third-party code is hard because of compiler setting variation. Thomas used hashing methods that preserve similarity between hashes in combination with deep learning. He talks about building the training set, the training, and provides examples of the results. Bottom line: simple string search is 90% as effective as this method.

Twinkly Twinkly Little Star

Mark Barnes analyzed the communication with smart Christmas lights reversed the firmware and used the endpoints to control the lights programmatically and make a snake game out of them. Along the way, found hardcoded credentials to the global MQTT server which can allow an attacker to power off all Christmas lights in the world potentially!

Abusing Mount Points over the SMB Protocol

A quick write-up of SMBv2 support of symlinks. The lesser known feature described here is the ability to follow links on the server if a mount point exists. The method could be interesting for red teamers, but what was most interesting to me was James’ thought process when researching this.

New Tools

Windows-centric x64 user-mode application emulator

A windows emulator POC by Alex Ionescu. Ionescu’s primary goal was to hook and debug system calls for malware analysis easily and to show how easy it is to spin-up an emulated environment without a guest OS image using new windows API introduced in Redstone 4 and Redstone 5 updates.

Windows Sandbox

A new sandbox by Microsoft that allows execution of untrusted code in an isolated and disposable environment. When the sandbox starts, it acts as a clean Windows installation, and when stopped everything is discarded. The post starts by presenting the features with a quick-start guide and then goes deeper into the internals of how the image is generated, how the memory is managed, how scheduling is done without disturbing the host, how such a light footprint is achieved through snapshotting, and more.

SNDBOX: AI Malware Analysis (on Hacker News) #madeinisrael

A malware analysis tool by Ariel Koren and Dr. Ran Dubin :) The tool fools the malware into thinking it is not running in a sandbox by creating wrappers around kernel functions. Even malware that sleeps before executing is tricked into executing immediately. It also has some unique features like RPC monitoring.

Pe-sieve

A process memory scanner that recognizes and dumps a variety of potentially malicious implants (replaced/injected PEs, shellcodes, inline hooks, hollowed processes, process doppelganging, and more).
